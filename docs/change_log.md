
# Change Log

프로젝트 방향, 버전 계획, 브랜치 전략, 파일 구조 같은 운영 기준이 바뀌었을 때 이유를 남기는 문서입니다.

## 기록 기준

- MVP 범위가 바뀐 경우
- 버전 목표가 바뀐 경우
- 브랜치 전략이 바뀐 경우
- 폴더 구조 또는 파일 분리 기준이 바뀐 경우
- 기술 스택이 바뀐 경우
- 기존 문서 기준과 다른 결정을 한 경우

## 기록 형식

```md
## YYYY-MM-DD - 변경 제목

- 변경 전:
- 변경 후:
- 이유:
- 영향:
- 관련 문서:
```
