
# Version Checklist

버전별 체크리스트는 `docs/versions/v*/checklist_*.md`에 생성합니다.

이 문서는 공통 체크리스트 기준입니다. 새 버전 폴더를 만들 때 이 항목을 복사해서 해당 버전에 맞게 수정합니다.

## 기획

- [ ] 사용자가 준 기획서를 확인했다.
- [ ] 현재 버전 목표가 `plan_*.md`에 정리되어 있다.
- [ ] 이번 버전에서 하지 않을 일이 명확하다.
- [ ] 기능 우선순위가 정리되어 있다.

## 브랜치

- [ ] 현재 기준 브랜치를 확인했다.
- [ ] `version/v*` 브랜치를 만들었다.
- [ ] 기능별 `feature/v*-*` 브랜치를 만들었다.
- [ ] 브랜치 흐름을 `branch_log_*.md`에 기록했다.

## 구현

- [ ] P0 기능이 구현되어 있다.
- [ ] 파일 구조가 `file_structure_rules.md` 기준과 맞다.
- [ ] 로딩 상태가 있다.
- [ ] 빈 상태가 있다.
- [ ] 에러 상태가 있다.
- [ ] 모바일 또는 작은 화면에서 사용할 수 있다.

## 검증

- [ ] 실행 확인:
- [ ] 빌드 확인:
- [ ] 테스트 확인:
- [ ] 수동 확인:
- [ ] 수행하지 못한 검증과 이유:

## 병합 전

- [ ] feature 브랜치 변경 범위를 확인했다.
- [ ] 관련 없는 변경이 포함되지 않았다.
- [ ] 기능별 커밋이 정리되어 있다.
- [ ] `version/v*` 병합 전 체크리스트를 확인했다.

## 버전 종료

- [ ] `review_*.md`에 완료 항목을 정리했다.
- [ ] 다음 버전으로 넘길 일을 정리했다.
- [ ] 사용자가 `develop` 대상 PR을 만들 수 있도록 PR 초안을 정리했다.
